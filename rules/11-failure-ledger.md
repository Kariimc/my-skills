# Failure Ledger — read before working, feed on every failure

`FAILURES.md` at this repo's root is the canonical cross-surface failure ledger:
every entry is a road already PROVEN dead (SYMPTOM → BANNED ROAD → THE ROAD THAT
WORKS). The live working copy on Kariim's machine is `~/.claude/FAILURES.md`;
this repo copy is what cloud and fresh surfaces read.

Two duties, same force as every other rule here:

1. **Read it before any work.** Repeating a banned road is a rule violation,
   not a judgment call.
2. **Feed it.** When any approach fails twice or burns more than 15 minutes,
   append a new `## F-NN` entry as part of the fix — never on request, never
   deferred. Keep `~/.claude/FAILURES.md` and this repo copy in sync in the
   same pass.

A ledger nobody appends to is dead weight. The entry IS part of the fix.
