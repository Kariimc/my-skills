# My Coding Journey — Command Line & Python

A gentle, hands-on plan to learn the command line and Python over about three
months. We go **one small lesson at a time**. No rushing, no jargon without a
plain-English explanation, and we build **real things** — not just dry exercises.

## How we work together
- **Short, hands-on lessons.** You type things on your own computer; I explain
  what each piece does and *why*.
- **You drive.** Say *"next lesson"* when you're ready, or *"I've only got 15
  minutes today"* and I'll size the lesson to fit.
- **Every lesson ends with** a quick *"What you just learned"* recap and an
  optional *"try it yourself."*
- **Mistakes are normal and welcome.** Red error text is just the computer
  asking a question — we read it together.

### Phase 1 — The Command Line on-ramp (Weeks 1–2)
Goal: feel at home in the terminal and never be scared of it again.
- [ ] Where am I? What's here? Moving around (`pwd`, `ls`, `cd`)
- [ ] Making and looking at files and folders
- [ ] Running a Python file from the terminal

### Phase 2 — Python Fundamentals (Weeks 3–8)
Goal: write small, useful programs on your own.
- [ ] Your first program (`print`)
- [ ] Variables — storing things
- [ ] Text, numbers, and true/false
- [ ] Asking the user for input
- [ ] Making decisions (`if` / `else`)
- [ ] Repeating things (loops)
- [ ] Lists — collections of things
- [ ] Functions — reusable building blocks
- [ ] Dictionaries — labeled data
- [ ] Reading and writing files
- [ ] Handling errors gracefully

### Phase 3 — Build Something Real (Weeks 9–12)
- [ ] Pick a project you actually care about
- [ ] Use libraries (other people's ready-made code)
- [ ] Build it together with Aider + Claude
- [ ] Put it on GitHub and share it

## Woven in throughout
Reading error messages calmly · using Google and docs · basic Git (saving your
work) · asking good questions.
