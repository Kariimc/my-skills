# Surface routing — do the work where the tools are (fires on cross-surface tasks)

Before doing (or refusing) work that touches another machine or surface, check
the environment fact sheet first — `.claude/env-facts.local.md` (written by
`hooks/env-scout.sh` at session start) and the environment facts in PROGRESS.md.

- **If THIS surface holds the needed channel, do the work here.** Writing a
  prompt for another surface while holding working tools is a banned road
  (ledger: the F-45 "prompts for another surface" entry).
- **If it genuinely needs another surface** (e.g. anything that must execute or
  install on the laptop — no cloud session can; the screen bridge is vision-IN
  only), send a precise, complete relay inbox message to that surface
  (`rules/08-relay.md`) — never a set of steps for Kariim to ferry.
- **Never assert "this surface can't" from memory.** The fact sheet is probed
  fresh each session; trust it over recollection, and update PROGRESS.md the
  same session a capability is gained or lost.
