# My Coding Journey — Command Line & Python

A gentle, hands-on plan to learn the command line and Python. We go **one small
lesson at a time**. No rushing, no jargon without a plain-English explanation,
and we build **real things** — not just dry exercises.

## 🎯 Your goal & pace
- **Goal: get a coding job.** Everything we do is aimed at making you *hireable*
  and *sought-after* — real, market-valued skills and a portfolio that proves
  them.
- **Pace: ~2 hours/week, taught as we go,** in tiny daily bites (10–20 min each).
- **Cheat sheets:** a growing set of quick-reference cards in `docs/cheatsheets/`
  covering exactly what employers expect you to know.
- **Head start:** you've already built 5–6 small projects and set up Git +
  GitHub automation. Most job applicants haven't shipped *anything* — so our job
  is to make sure you deeply *understand* what you build and can explain it.

> Honest note on timeline: at ~2 hrs/week, "fully job-ready" is more likely a
> 6–12 month arc than "a few months." The single biggest lever is **hours** —
> even bumping to 4–5 hrs/week dramatically speeds things up. Either way, we aim
> every minute at high-value skills + a strong portfolio.

## The skills that make you hireable (our north star)
1. Python you genuinely understand (not just AI-generated)
2. Git & GitHub — saving and sharing work ✅ *(started)*
3. Command-line comfort ✅ *(done today)*
4. Reading & fixing error messages calmly
5. SQL — talking to databases (in huge demand)
6. 2–3 polished portfolio projects you can explain line by line
7. The basics of how the web & APIs work
8. (For data/analysis) pandas — crunching spreadsheets with code

## How we work together
- **Short, hands-on lessons.** You type on your own computer; I explain the *why*.
- **You drive.** Say *"next lesson"*, or *"I've only got 15 minutes today."*
- **Every lesson ends with** a *"What you just learned"* recap + optional
  *"try it yourself."*
- **Mistakes are welcome.** Red error text is the computer asking a question —
  we read it together.
